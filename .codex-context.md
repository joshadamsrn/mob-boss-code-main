## Known Constraints and Conventions
- Follow `CONVENTIONS.md` (ports/adapters, DTO parsing via `from_payload`, domain logic in `src/`, Django at edges).
- `rooms` owns pre-game/lobby; `gameplay` owns in-round lifecycle/trial/economy behavior.
- Prefer targeted tests for edited area before broad runs.
- Keep player-role leakage blocked in non-moderator projections.
- Raw player IDs are acceptable for hidden form values / internal action payloads, but not for visible user text.
