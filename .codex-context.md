  - sale offers require buyer accept/decline prompt,
  - insufficient-funds on accept only notifies buyer; merchant sees decline,
  - merchant can sell item back to central supply for 90% of acquisition.
- Participant badge update:
  - accused during `trial_voting` now displays `On Trial`.

  - ledger entry model added to gameplay snapshots and sqlite persistence,
  - ledger entries recorded for supply purchases, supply buybacks, accepted money gifts,
    murder transfers, and conviction transfers,
  - strict reconciliation re-enabled so money cannot silently leave the game in implemented flows,
  - death flow preserves murdered-player resources on the dead participant until a real recipient exists.
- Phase B completed:
  - EFJ hook points added to guilty-verdict resolution,
  - EFJ item auto-consumes on guilty verdict,
  - accused remains active after EFJ auto-use,
  - EFJ purchase value transfers from the central-supply ledger holder to a police recipient,
  - recipient only gets a private bribe notification,
  - no public recipient leak.
- Phase C completed:
  - vest-blocked gunshots now consume the vest,
  - no trial starts when a vest blocks a reported gunshot,
  - vest value transfers from `central_supply` to the attacker to preserve reconciliation,
  - attacker identity stays hidden from the public vest notice,
  - murder reports now require `attack_classification`,
  - public murder notice includes weapon display name.
- Role-ability UI baseline completed:
  - alive participants now always get a role-ability card,
  - `Don`, `Under Boss`, and `Kingpin` keep actionable controls,
  - passive/automatic roles now show specific “current hooks” details,
  - placeholder roles now show role-specific summary/detail blocks instead of generic filler.
- Deputy activated power implemented:
  - `Protective Custody` (once/game, information phase),
  - target alive/non-self, 5-minute immunity window,
  - attempted murder still starts trial but target remains alive with no transfer,
  - timer and notices visible to moderator + deputy + target.
- Sheriff activated power implemented:
  - `View Jury Log` (twice/game, any in-progress phase),
  - reveals most recent jury roster only (no votes),
  - fails with `No jury history yet` before first jury,
  - sheriff reveal lasts 60 seconds; moderator has persistent latest-jury visibility.
- Captain activated power implemented:
  - `Asset Freeze` (once/game, any in-progress phase),
  - freezes one alive non-self target for 10 minutes,
  - blocks money actions (buy/sell/gift send/receive),
  - cancels pending sale/gift/money offers involving target and notifies involved users,
  - timer visible to moderator + captain + target.
- Lieutenant activated power implemented:
  - `Information Briefcase` (once/game, any in-progress phase),
  - privately reveals alive faction counts (Police/Mob/Merchant) to Lieutenant,
  - reveal lasts 60 seconds with no countdown.
- Sergeant activated power implemented:
  - `Capture` (once/game, information phase only),
  - one active captured player at a time (alive non-self target),
  - captured player is blocked from actions/powers and cannot be murder target while captured,
  - timer visible to moderator + sergeant + captured player,
  - auto-release branch if required leader responsibilities cannot be reassigned.
- Detective activated power implemented:
  - `Investigation` (once/game, any in-progress phase),
  - can target any participant, including alive, dead, or jailed players,
  - reveal visible to Detective only for 60 seconds with no countdown,
  - shows the target's last 3 player-to-player transactions only,
  - historical sale + money-gift events backfill from ledger where available,
  - accepted sales, money gifts, and item gifts now persist as player transaction history.
- Inspector activated power implemented:
  - `Record Inspection` (once/game, any in-progress phase),
  - can target one dead or jailed participant except self,
  - reveal visible to Inspector only for 60 seconds with no countdown,
  - shows only the target's role name,
  - fails with `No jail or morgue records available yet.` when no eligible targets exist.
- Police Officer activated power implemented:
  - `Confiscation` (once/game),
  - can be armed during `information`, `accused_selection`, or active `trial_voting` before all votes are submitted,
  - applies to the next guilty verdict only,
  - if the officer is still alive and EFJ does not trigger, jailed inventory is liquidated to cash and redistributed between the Police Officer and other alive police,
  - if the officer is inactive, EFJ triggers, the verdict is not guilty, or no recoverable resources exist, the armed effect is still consumed and resolved via notices,
  - normal single-recipient conviction inheritance is overridden when confiscation succeeds.
- Cop automatic power implemented:
  - auto-protects once/game when alive non-jailed player count reaches 3 or fewer,
  - system-generated bulletproof vest is added if Cop does not already have one,
  - if Cop already has a vest, the trigger is still consumed and only notices fire,
  - only Cop and moderator are notified,
  - status-only card shows `Active` / `Inactive`.
- Enforcer automatic power implemented:
  - first successful reported murder grants a one-time 50% cash bonus based on victim transferred money only,
  - bonus rounds up to the next `$10`,
  - bonus is system-generated and ledgered from `central_supply`,
  - zero-cash kill still consumes the power,
  - only Enforcer and moderator are notified,
  - status-only card shows `Active` / `Used`.
- Made Man activated power implemented:
  - `Skip Middle Man` (once/game),
  - may buy one currently active central-supply item at the live supply price during any `in_progress` phase,
  - still pays with own money and remains blocked by freeze/capture/death/jail rules,
  - insufficient funds do not consume the power,
  - moderator + active merchant-role players + Made Man are notified with burglary flavor text / confirmation,
  - actionable role card is live.
- Merchant activated power implemented:
  - `Wholesale Order` (once/game, information phase only),
  - only the base `Merchant` role gets this power,
  - buys one active central-supply item at a 30% discount from the listed price,
  - discounted price uses the normal nearest-$10 rounding,
  - purchase still consumes shared stock and is ledgered from player to `central_supply`,
  - insufficient funds do not consume the power.
- Arms Dealer automatic power implemented:
  - `Starting Gun Cache` (automatic at game launch),
  - Arms Dealer starts with one Tier 1 gun in inventory,
  - that gun is removed from shared central supply stock at session start,
  - only the Arms Dealer receives the startup notification,
  - no manual activation card action is required.
- Smuggler activated power implemented:
  - `Smuggle` (once/game, information phase only),
  - targets one alive non-self player who is not currently captured,
  - steals one truly random eligible item from the target's current inventory on the server,
  - items already tied up in pending sale/gift offers are ineligible,
  - if no eligible item exists, the power is still consumed,
  - private result notices go to Smuggler, target, and moderator only,
  - successful theft preserves acquisition value/resale price and is stored in Detective-visible transaction history as `Stolen Item`.
- Gun Runner activated power implemented:
  - `Charisma` (once/game, information phase only),
  - activation starts a 3-minute timer visible on the Gun Runner role card,
  - every accepted player-to-player sale during the active window pays an extra 30% system bonus based on the accepted sale price,
  - bonus rounds to the normal nearest-$10 rule,
  - bonus is generated from `central_supply` and does not charge the buyer extra,
  - only Gun Runner and moderator are notified on activation and each payout,
  - frozen accounts do not block activation, but still block sale completion while the timer runs,
  - captured state blocks activation.
- Central supply stock behavior changed:
  - active catalog items are now shared one-time stock globally,
  - when a supply item is bought, it becomes unavailable to everyone,
  - when a merchant sells an item back to central supply, that catalog item becomes active again and is available for purchase,
  - Made Man still cannot sell to supply because sell-back remains merchant-only.
- Gangster activated power implemented:
  - `Tamper` (once/game, one trial only),
  - available during active `trial_voting` after jury selection and before effective vote completion,
  - targets one current juror; the juror remains visibly on the jury but their counted vote is suppressed for that trial,
  - Gangster gains a separate replacement vote slot with its own fresh 10-second timer,
  - if Gangster is already a juror, Gangster may cast both the normal jury vote and the tamper vote,
  - if the targeted juror already voted, that counted vote is removed immediately,
  - if Gangster dies, is jailed, or is captured before resolution, tamper cancels and the juror's vote counts normally again,
  - silencing does not cancel tamper,
  - only Gangster, targeted juror, and moderator are notified; moderator trial UI also shows the tampered juror.
- Street Thug activated power implemented:
  - `Steal` (once/game),
  - available during any in-progress phase,
  - targets one alive non-self participant who is not currently captured,
  - if target has at least `$100`, exactly `$100` moves from target to Street Thug,
  - if target has less than `$100`, no money moves and the power is still consumed,
  - Street Thug, target, and moderator are notified of the result,
  - successful steals are persisted in the gameplay ledger for reconciliation.
- Ghost View implemented:
  - dead or jailed players get a `Ghost View` banner and moderator-style information visibility,
  - ghost viewers are warned not to speak to active players or show their device,
  - ghost view is informational only and does not expose moderator mutation controls.
- Felon automatic power implemented:
  - `Escape` (automatic on jail),
  - when a `Felon` is jailed, a 30-minute sit-out timer starts,
  - Felon and moderator are notified that the Felon must sit out and cannot talk to active players or view their devices,
  - while the timer is active, the jailed Felon does not receive Ghost View,
  - if the game is still in progress when the timer expires, the Felon returns from `jailed` to `alive`,
  - re-entering Felon balance is reset to `$10`,
  - public notice announces that a felon escaped and is back in the game.
- Trial-voting timeout behavior clarified/implemented:
  - when vote clocks expire, the trial resolves using only counted votes submitted before expiry,
  - missing juror votes count as no vote,
  - guilty requires strictly more counted `guilty` votes than counted `innocent` votes,
  - ties resolve as `innocent` / not guilty.
